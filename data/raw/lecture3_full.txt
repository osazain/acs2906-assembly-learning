Floating Point
17
Finite-length encodings cannot represent numbers such as
[x]
1/3 and 5/7
Fractional binary notation can only represent numbers in the
[x]
form V = M × 2y
Examples:
[x]
5 × 2-3 = 5/8 = 0.101
[x]
5 × 2-4 = 5/16 = 0.0101
[x]
5 × 2-8 = 5/256 = 0.00000101
[x]
Notice: y determines location of point
[x]
Only need to store M and y
[x]
Hence: floating point notation

Normalized Values
21
Most common case
[x]
Occurs when exp is neither all 0s or all 1s
[x]
i.e. 0 < exp < 2k - 1
[x]
E is interpreted as representing a signed integer in
[x]
biased form
E = exp - bias
[x]
exp is an unsigned number
[x]
k-1
Bias = 2 - 1 (127 or 1023 for single and double)
[x]

Normalized Values Continued
22
Bits in frac are interpreted as 0.f ... f f
[x]
n-1 1 0
And M is interpreted as 1 + frac
[x]
This is called an implied leading 1 because M can be
[x]
viewed as 1.f ... f f
n-1 1 0
Trick for getting an extra bit of precision for free
[x]

Denormalized Values
23
Occurs when exp is all 0s
[x]
i.e. exp = 00 ... 0
[x]
E is interpreted as E = 1 - bias
[x]
M is interpreted as frac without implied leading 1
[x]
i.e. M = 0.f ... f f
[x]
n-1 1 0

Denormalized Values Continued
24
Why use denormalized values ?
Represent 0 (not possible with normalized values)
[x]
M >= 1 and (-1)s × M × 2E
[x]
Represent numbers very close to 0
[x]
Provides a property called gradual underflow in which
[x]
possible numeric values are spaced evenly near 0

Floating Point Subtraction
36
Like addition in terms of aligning radix points
[ ]
But, requires a few more rules:
[ ]
Pay attention to the signs and magnitudes of
[ ]
operands. We always want the value of M to be a

Sign of the result after subtraction is encoded by
[ ]
