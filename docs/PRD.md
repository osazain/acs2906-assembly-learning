# PRD - ACS2906 Learning Platform

## Vision
Create a source-grounded, mastery-based learning platform for ACS2906 that transforms lectures, examples, worksheets, assignments, and exam review materials into a beautiful interactive study system.

## Core User
A student studying 8086 assembly who wants to understand concepts deeply, practice effectively, identify weaknesses, and prepare for assignments and exams across devices.

## Product Pillars
1. Learn from real course material
2. Practice with high-quality, source-linked interactions
3. Diagnose weakness precisely
4. Improve through targeted remediation
5. Look and feel elegant on desktop and phone

## Must-Have Experiences
- Course Atlas: an overview of the whole course with linked lectures, examples, worksheets, assignments, and review material
- Lecture Reader: each lecture broken into sections with inline examples and quick checks
- Example Explorer: browse examples by lecture, instruction, concept, difficulty, and assignment relevance
- Drill Mode: focused practice by topic or weakness
- Diagnostic Mode: adaptive-ish testing that identifies missing understanding
- Game Mode: source-grounded mini-games for instruction recognition, tracing, control flow, stack/procedure logic, and debugging
- Assignment Trainer: assignment-like practice with hints and worked comparisons
- Exam Mode: timed mixed-topic practice tests
- Mastery Dashboard: heatmaps and review recommendations
- Simulator Integration: run snippets or guided steps in-browser

## Quality Bar
- Every major topic in the course should have at least one learning path and one practice path.
- Every assessment result should tell the learner exactly what to review next.
- The UI should look polished enough that it feels like a premium study product rather than a class project.

## Out of Scope for MVP
- Multi-user accounts
- Teacher/admin portal
- Server-required personalization
- Real-time collaboration
- OCR-heavy ingestion workflows

## Success Criteria
- The app covers all lectures and maps examples correctly.
- Users can navigate from mistake -> explanation -> source example quickly.
- GitHub Pages deployment works reliably on phone.
- Local progress persists across sessions in the same browser.
